# Upwork / Fiverr Milestone Structure (Anti-Shallow)

This is critical — this is how you avoid paying for fluff.

## Milestone 1 — Setup & Understanding (10–15%)

### Deliverables

- Repo runs locally
- Harness works
- Initial prompt improvements
- 2 example outputs reviewed with you

➡️ **Payment released only after review**

## Milestone 2 — Core Capability Coverage (40–50%)

### Deliverables

- Majority of golden prompts implemented
- Core scenarios covered
- Structure consistently enforced

### Gate

- ≥80% outputs meet rubric
- No generic responses

## Milestone 3 — Edge Cases & Quality (25–30%)

### Deliverables

- Missing-context handling
- Tradeoff-heavy prompts
- Safety cases (Coach)

### Gate

- Tests added
- Reviewer-approved outputs

## Milestone 4 — Polish & Handoff (10–15%)

### Deliverables

- docs/examples.md
- README updates
- Final review fixes

## Payment rule (important)

**No milestone is approved based on "looks good" alone.**

Approval requires:

- passing tests
- reviewed example outputs
- rubric score ≥ threshold
